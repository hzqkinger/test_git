##### 文件状态 #######
工作目录下的文件都只有两种状态：已跟踪和未跟踪
 已跟踪的文件指的是已经被纳入版本控制的文件，它们可能是为更新、已修改或者已放入暂存区
 而其他文件都是未跟踪文件，指的是未被纳入版本控制的文件

初次从远程仓库中克隆到工作目录的文件都属于已跟踪未修改，经过某些编辑后，变为已跟踪已修改，然后将它们放到暂存区域，等待最后一次性提交暂存区域里所有的文件。

git status   可以查看当前工作目录所有文件的状态
	1，在暂存区以备提交的更新：
		git rm --cached <file>	把某个文件从暂存区中删除，变成未跟踪文件
		git rm -f <file>		强制删除某个文件
	2，未在暂存区以备提交的更新：
		git add <file>			把该文件放到暂存区
		git checkout <file>		放弃对某个文件的更新
	3，未跟踪文件：
		git add <file>			把该文件放到暂存区
